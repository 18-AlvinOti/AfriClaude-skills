## 🧰 The Toolkit: Core Skills

This collection evolves constantly, but here are the core capabilities currently deployed in my workflow:

*   **`quant-auditor-skills.skill`**: My primary quantitative auditor. This skill focuses on numerical and economic attack vectors within DeFi protocols—analyzing slippage, fee structures, and incentive mechanisms for logical flaws.
*   **`pentest-auditor.skill`**: A specialized penetration testing mindset. It guides Claude to think like an adversary, probing for traditional security weaknesses in Web3 infrastructure and dApps.
*   **`certora-cvl-spec-writer.skill`**: Automates and assists in the creation of Certora Verification Language (CVL) specifications. Translates complex invariant ideas into formal verification code, bridging the gap between intention and proof.
*   **`erc-interface-scanner.skill`**: A rapid-assessment tool that scans and verifies ERC standard implementations against best practices, identifying deviations and potential compliance issues in contract interfaces.
*   **`bug-bounty-program-manager.skill`**: Streamlines the chaos of bug hunting. Helps triage findings, draft professional reports, and manage communications, ensuring zero signal is lost in the noise.

## 🛠️ How to Use

These are `.skill` files. To utilize them:

1.  **Save the File**: Download the raw `.skill` file you wish to use.
2.  **Import into Claude (or compatible interface)**: Use the project or knowledge base feature to upload the file as a core instruction set or context document.
3.  **Engage**: Begin your session by referencing the skill. For example: *"Activate the `quant-auditor-skills.skill`. We need to analyze the new lending protocol's liquidation mechanism."*
